# Contributing to ReactiveShield

Thank you for your interest in contributing to ReactiveShield! Here's how to get involved.

## How to Contribute

### Reporting Bugs

1. Check the [Issues](https://github.com/suneetbhardwaj-art/reactiveshield/issues) page to avoid duplicates.
2. Open a new issue with:
   - A clear title and description
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Features

1. Open an issue with the label `enhancement`.
2. Describe the feature, its use case, and why it would benefit the project.

### Submitting Code

1. **Fork** the repository
2. **Create** a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make** your changes, following the existing code style
4. **Commit** with a clear message:
   ```bash
   git commit -m "feat: describe what you added"
   ```
5. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open** a Pull Request against the `main` branch

## Code Style

- Use **TypeScript** with strict typing
- Follow the existing component structure in `src/components/`
- Use **Tailwind CSS** classes for styling — avoid inline styles
- Keep components small and focused (single responsibility)
- Run `npm run build` before submitting to ensure no build errors

## Commit Message Format

Use conventional commit prefixes:

| Prefix | Purpose |
|---|---|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `docs:` | Documentation changes |
| `style:` | Styling/UI changes |
| `refactor:` | Code restructuring |
| `chore:` | Build/config changes |

## Code of Conduct

Be respectful and constructive. All contributors are expected to maintain a welcoming and collaborative environment.
